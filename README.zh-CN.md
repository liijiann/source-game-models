# Source Game Models

[English](README.md) | **简体中文**

`source-game-models` 是一个面向多种 AI Agent 的 Web 3D 建模 skill，核心方法是“先参考，再建模”。它引导 AI 在确定最终建模方案前研究优秀的现有模型，避免默认用简陋的基础几何体拼装成品。

![基础几何体与参考成熟模型后进行运行时改造的骷髅对比](preview.png)

预览图在相同镜头和光照下展示同一题材。左侧是基础几何体拼装的 baseline；右侧参考成熟模型，并针对目标场景调整了比例、材质、部件和动画。参考模型为 [Quaternius 制作的 Skeleton](https://poly.pizza/m/DM4QScSmbS)，采用 [CC0 1.0](https://creativecommons.org/publicdomain/zero/1.0/) 授权。

## 目的

这个 skill 的目标是帮助 AI 提升建模质量，而不是单纯下载角色或道具。它会引导 AI：

- 从现有模型中研究轮廓、比例、材质语言、细节密度、结构、骨骼和动画。
- 把参考分析转化为适合当前项目的具体建模与改造方案。
- 根据项目的美术方向和技术约束，对资产进行适配、重组、组合或重建。
- 只把基础几何体用于原型调试或有意采用的抽象风格，不默认当作最终成果。

模型搜索和下载只是辅助步骤。找到的资产应当作为合法的建模参考或改造基础，不能未经分析便原样放进项目。

## 安装

通过你的 AI 编程 Agent 所支持的自定义 skill 机制添加 [`source-game-models/`](source-game-models/) 文件夹。该文件夹是完整独立的，其中包含 skill 指令、模型来源指南、Web 3D 质量指南和 glTF/GLB 检查脚本。

## 使用

让 Agent 通过参考分析来改进建模：

```text
使用 source-game-models skill 改进我的 Three.js 游戏主角。寻找授权合规的优秀模型作为参考，分析它们在轮廓、比例、材质、骨骼和动画方面值得借鉴的地方，再适配或重建出符合当前项目风格的结果。
```

支持自动发现 skill 的 Agent，也可以在处理 Three.js、React Three Fiber 或 Babylon.js 项目的角色、道具、载具、建筑、环境、生物、骨骼或动画时自动启用它。

## 工作流程

1. 理解项目的视觉方向、尺度、材质、细节等级、动画需求和性能预算。
2. 寻找少量高质量且授权允许使用的参考模型。
3. 比较每个参考能为建模方案提供什么，而不只是查看缩略图是否好看。
4. 下载或接入资产前，先让用户确认参考方向。
5. 把选定参考转化为针对当前项目的比例、材质、部件、拓扑、骨骼或动画改造。
6. 在 `THIRD_PARTY_ASSETS.md` 中记录来源、许可证和修改内容。
7. 在真实游戏场景中验证构图、动画、渲染效果和移动端性能。

## 谨慎使用参考

“先参考，再建模”不代表盲目照搬。这个 skill 会拒绝授权不明、盗提、仅限非商业用途或禁止衍生修改的资产，保留出处与署名记录，也不会把未经修改的下载模型包装成已经完成的建模成果。
